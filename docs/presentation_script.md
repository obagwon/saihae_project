# 사이해 발표 영상 대본 및 시연 자료

> 권장 발표 시간: 7분 내외  
> 발표 흐름: 앱 소개 → 구현 기능 → 주요 기술 → 데모 시연 → 개발 소감

## 0. 발표 준비 체크리스트

- 휴대폰 에뮬레이터 또는 실제 기기에서 앱을 실행해 둡니다.
- 발표 시작 전 화면은 앱 첫 화면 또는 홈 화면으로 맞춥니다.
- 데모 중 보여줄 순서는 `초기 화면 → 홈 → 기록 → 성향 테스트 → 나 분석 → 관계 → 허브티 → 다크모드`를 추천합니다.
- 말이 빨라지면 5분 안팎, 기능을 하나씩 천천히 보여주면 8~10분 정도가 됩니다.

## 2. 5~7분용 발표 대본

### 2-1. 오프닝 및 앱 소개 (0:00~0:40)

안녕하세요. 제가 개발한 앱은 **사이해**입니다.  
사이해는 “사이를 이해하다”와 “나를 이해하다”라는 의미를 담은 정서적 자기이해, 관계 웰니스 앱입니다.

이 앱은 전문 심리 진단 앱이라기보다는, 사용자가 오늘의 감정을 가볍게 기록하고, 자신의 관계 성향을 돌아보며, 관계를 조금 더 부드럽게 이해할 수 있도록 돕는 앱입니다.

화면 분위기는 너무 딱딱한 진단 앱처럼 보이지 않도록 따뜻한 색감과 카드형 UI를 중심으로 구성했습니다.

**이때 보여줄 것**

1. 앱 실행 직후 첫 화면을 보여줍니다.
2. 1~2초 뒤 홈 화면으로 넘어가는 흐름을 보여줍니다.
3. 홈 화면의 “오늘의 마음 체크” 영역을 가리키며 앱 목적을 설명합니다.

### 2-2. 구현 기능 설명 (0:40~2:30)

사이해의 주요 기능은 크게 다섯 가지입니다.

첫 번째는 **홈 화면**입니다.  
홈에서는 오늘의 마음을 바로 체크할 수 있고, 성향 테스트나 관계 비교 같은 핵심 기능으로 빠르게 이동할 수 있습니다. 이미 오늘 감정을 기록했다면 최근 기록도 확인할 수 있게 구성했습니다.

두 번째는 **감정 기록 기능**입니다.  
사용자는 감정 이모지를 선택하고, 감정 강도를 1단계부터 5단계까지 고른 뒤, 필요하면 한 줄 메모를 남길 수 있습니다. 기록한 내용은 목록으로 다시 볼 수 있고, 최근 기록을 기반으로 통계와 추천 문구도 확인할 수 있습니다.

세 번째는 **성향 테스트 기능**입니다.  
총 12개의 질문에 답하면 사용자의 관계 성향을 8가지 유형 중 하나로 보여줍니다. 각 결과에는 성향 이름, 설명, 강점, 주의할 점, 관계 스타일, 추천 행동이 포함되어 있습니다.

네 번째는 **나 분석 탭과 관계 가이드**입니다.  
나 분석 탭에서는 저장된 성향 결과와 점수 요약을 확인할 수 있고, 관계 탭에서는 내 성향에 맞는 관계 팁을 보거나 상대 성향을 선택해서 비교할 수 있습니다. 이때 “좋다/나쁘다”처럼 단정하기보다는 서로의 표현 방식과 속도 차이를 이해하도록 문구를 구성했습니다.

다섯 번째는 **허브티 추천 기능**입니다.  
최근 감정 기록을 바탕으로 현재 마음 상태에 어울리는 허브티를 추천합니다. 캐모마일, 페퍼민트, 루이보스, 라벤더 같은 허브티를 상황별로 제안하고 주의사항도 함께 보여줍니다.

**이때 보여줄 것**

1. 하단 탭 5개를 순서대로 짧게 누릅니다: 홈 → 나 분석 → 관계 → 기록 → 허브티.
2. 각 탭에서는 오래 머무르지 말고 “이런 기능이 있습니다” 정도로만 소개합니다.
3. 자세한 조작은 뒤의 데모 시연 구간에서 보여준다고 말합니다.

### 2-3. 주요 기술 적용 설명 (2:30~4:00)

기술적으로는 **Flutter와 Dart**를 사용해서 앱을 구현했습니다. Flutter의 장점인 위젯 기반 구조를 활용해 홈, 기록, 분석, 관계, 허브티 화면을 각각 분리했고, 하단 탭으로 화면을 전환하도록 만들었습니다.

UI는 Material Design을 기반으로 만들었고, 앱의 분위기에 맞게 색상 팔레트, 카드 모서리, 그림자, 버튼 스타일을 공통 테마로 정리했습니다. 그래서 화면마다 비슷한 감성을 유지할 수 있었습니다.

데이터 저장은 서버나 로그인 없이 **shared_preferences**를 사용했습니다. 사용자의 감정 기록과 성향 테스트 결과를 기기 내부에 저장하기 때문에 앱을 껐다 켜도 결과가 유지됩니다.

또 하나 신경 쓴 부분은 **다크모드**입니다. 시스템 설정에 따라 라이트 모드와 다크 모드가 자동으로 적용되도록 했고, 배경색, 카드색, 글자색이 다크모드에서도 자연스럽게 보이도록 테마를 따로 구성했습니다.

마지막으로 이미지 에셋도 사용했습니다. 시작 화면 이미지와 8가지 성향 카드 이미지를 넣어서 결과 화면과 관계 화면이 더 완성도 있게 보이도록 했습니다.

**이때 보여줄 것**

1. 화면 하단 탭 전환이 부드럽게 되는 모습을 보여줍니다.
2. 이전에 저장된 감정 기록 또는 성향 결과가 남아 있는 모습을 보여주며 로컬 저장을 설명합니다.
3. 가능하면 기기 다크모드를 켜서 앱 색상이 바뀌는 모습을 보여줍니다.

### 2-4. 데모 시연 (4:00~6:30)

이제 실제 앱 사용 흐름을 보여드리겠습니다.

먼저 홈 화면에서 오늘의 마음을 선택해 보겠습니다. 예를 들어 “피곤해요” 또는 “편안해요” 같은 감정을 선택하고 기록 탭으로 이동합니다.

기록 화면에서는 감정 강도를 선택하고, 메모에는 “오늘 발표 준비를 해서 조금 긴장됐다”처럼 짧게 작성한 뒤 저장합니다. 저장 후에는 방금 남긴 기록이 목록에 추가되는 것을 확인할 수 있습니다.

다음은 성향 테스트입니다. 테스트를 시작하면 12개의 질문이 나오고, 각 질문에 4단계로 답변할 수 있습니다. 발표 시간상 모든 문항을 자세히 읽기보다는 빠르게 선택해서 결과 화면으로 넘어가겠습니다. 결과 화면에서는 제 성향 이름과 설명, 강점, 주의할 점, 추천 행동을 확인할 수 있습니다.

이 결과는 나 분석 탭에도 저장됩니다. 나 분석 탭으로 이동하면 방금 나온 성향 결과와 점수 요약이 표시됩니다.

다음으로 관계 탭을 보겠습니다. 여기서는 내 성향을 기준으로 관계 가이드를 확인할 수 있고, 상대 성향을 선택하면 두 성향의 차이와 조심하면 좋은 부분을 비교할 수 있습니다. 이 기능은 관계를 맞다, 틀리다로 판단하는 것이 아니라 서로를 이해하는 방향으로 만들었습니다.

마지막으로 허브티 탭입니다. 방금 감정을 기록했기 때문에 앱이 현재 마음 상태에 맞는 허브티를 추천합니다. 추천 이유와 주의사항도 같이 보여주기 때문에 감정 기록이 간단한 자기돌봄 루틴으로 이어지도록 구성했습니다.

**이때 보여줄 것**

1. 홈에서 감정 하나 선택 후 기록 탭으로 이동합니다.
2. 기록 화면에서 강도와 메모를 입력하고 저장합니다.
3. 성향 테스트를 빠르게 진행하거나 이미 저장된 결과를 보여줍니다.
4. 나 분석 탭에서 저장된 결과를 확인합니다.
5. 관계 탭에서 상대 성향을 하나 선택해 비교합니다.
6. 허브티 탭에서 감정 기반 추천을 확인합니다.

### 2-5. 개발 소감 및 마무리 (6:30~7:30)

이번 앱을 개발하면서 가장 중요하게 생각한 점은 기능을 많이 넣는 것보다, 사용자가 부담 없이 자기 감정을 돌아볼 수 있는 흐름을 만드는 것이었습니다.

어려웠던 점은 여러 화면에서 같은 데이터를 자연스럽게 공유하는 부분이었습니다. 예를 들어 홈에서 감정을 선택하면 기록 탭으로 연결되고, 기록한 감정은 다시 허브티 추천에 영향을 주어야 했습니다. 또 성향 테스트 결과도 결과 화면, 나 분석 탭, 관계 탭에서 함께 사용되어야 해서 데이터 저장 구조를 고민하게 되었습니다.

개발하면서 Flutter의 화면 분리 방식, 상태 관리, 로컬 저장, 테마 구성에 대해 더 이해할 수 있었습니다. 특히 라이트 모드와 다크모드를 모두 고려하면서 색상과 UI 일관성을 맞추는 것이 생각보다 중요하다는 점을 배웠습니다.

앞으로 개선한다면, 감정 기록을 달력 형태로 보여주거나, 기록이 쌓였을 때 주간 리포트를 제공하고 싶습니다. 또한 사용자가 직접 목표나 자기돌봄 루틴을 설정할 수 있는 기능도 추가해 보고 싶습니다.

이상으로 사이해 앱 발표를 마치겠습니다. 감사합니다.

**이때 보여줄 것**

1. 홈 화면이나 나 분석 화면처럼 앱의 대표 화면을 띄워둡니다.
2. 마지막에 다크모드 화면을 보여주면 시각적으로 마무리가 좋습니다.

## 3. 10분까지 늘리고 싶을 때 추가할 내용

- 성향 테스트의 8가지 결과 이름을 2~3개 예시로 소개합니다.
- 감정 기록 통계가 어떤 의미인지 설명합니다.
- `shared_preferences`를 선택한 이유를 설명합니다: 로그인 없이 간단한 개인 기록 앱에 적합하고, 구현 범위가 프로젝트 규모에 맞았다고 말하면 좋습니다.
- 앱 용량 최적화에서 이미지 에셋이 큰 비중을 차지했다는 점을 언급합니다.
- 다크모드 구현 전후를 비교해서 UI 완성도를 강조합니다.

## 4. 발표자가 외우기 쉬운 핵심 문장

- “사이해는 나를 이해하고 관계를 이해하기 위한 정서적 자기이해 앱입니다.”
- “전문 진단보다는 사용자가 부담 없이 오늘의 마음을 기록하는 경험에 초점을 맞췄습니다.”
- “감정 기록, 성향 테스트, 관계 가이드, 허브티 추천이 하나의 자기돌봄 흐름으로 연결되도록 만들었습니다.”
- “데이터는 shared_preferences를 사용해 기기 내부에 저장되도록 구현했습니다.”
- “관계 기능은 잘 맞고 안 맞고를 판단하기보다, 서로의 표현 방식과 속도 차이를 이해하도록 설계했습니다.”
- “이번 프로젝트를 통해 Flutter의 화면 구성, 로컬 저장, 테마 관리, 다크모드 적용을 실습할 수 있었습니다.”

## 5. 촬영 팁

- 처음부터 완벽하게 말하려고 하지 말고, 위 시간표를 옆에 띄워두고 진행합니다.
- 기능 설명은 길게 말하기보다 화면을 보여주면서 짧게 말하는 편이 자연스럽습니다.
- 테스트 문항은 발표 중 모두 자세히 읽지 말고 빠르게 넘겨도 됩니다.
- 실수했을 때는 다시 처음부터 찍기보다, 잠깐 멈추고 “다시 이어서 설명드리면”이라고 말한 뒤 계속 진행해도 괜찮습니다.
- 5분 안에 끝내야 한다면 데모에서 성향 테스트 전체 진행 대신 저장된 결과 화면을 바로 보여주는 방식을 추천합니다.
